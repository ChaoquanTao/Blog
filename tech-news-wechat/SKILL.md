# 科技要闻公众号 Skill

把一周科技新闻做成可直接发布的微信公众号图文，含三部分产物：

1. **正文**：`.md`（内容底稿）+ 微信兼容 `.html`（排版稿）
2. **封面**：`.svg`（矢量源）+ `.png`（公众号可直接上传，2.35:1）

```
① 选题 & 核实  →  ② Markdown 底稿  →  ③ 微信 HTML 排版  →  ④ 封面图
```

---

## ① 选题与内容打磨

**先问清楚**（用 AskUserQuestion）：时间范围（本周/两周）、条数（建议 5–8 条）、是否有必须包含的指定新闻、读者偏好（纯科技 / 含财经/数码）。

**选题原则：**
- **时效第一**：只选本周/近两周真正发生的事。动手前用 WebSearch 核实日期，过期旧闻（哪怕很火）要替换或明确标注发布时间。
- **指定新闻必须核实**：用户给的新闻（如"某模型发布""某公司上市"）先 WebSearch 求证，常有偏差（例：是"美国政府叫停"而非公司主动下线）。以官方声明/权威媒体为准。
- **国内外搭配**：混入国产/中国市场相关条目（如国产芯片 IPO、国产大模型、苹果中国区政策），更贴近读者。
- **每条短小**：公众号 90% 在手机看。每条 = 一句标题 + 2–4 句正文，关键数字点出来即可。
- **加"料"**：给重点条目配 **Tips / 冷知识 / 背景** 小框（如"史上最大 IPO 纪录变迁"），提升信息密度和趣味。

**核实清单**：人物身价、IPO 金额、估值、纪录类说法（"史上最大""首个"）、政策生效范围——这些最易错，逐条搜证。回复用户时附 Sources 链接。

**底稿格式**：先写一个干净的 `.md`，每条用 `**N. 标题**` + 正文段落，便于用户审阅和后续多轮修改。改稿时只动对应条目。

## ② 微信公众号排版（轻松卡片风）

用户用 Chrome 打开生成的 HTML → Ctrl+A → Ctrl+C → 粘贴进 mp.weixin.qq.com 编辑器。浏览器复制的是渲染后富文本，微信保留 inline style。

完整可复用模板见 **`assets/wechat-template.html`**，直接改文案即可。下面是要点。

### 硬约束（踩过的坑）
- **必须有 `<!DOCTYPE html>` + `<head><meta charset="UTF-8"></head>`**，否则 Chrome 打开中文全是乱码（这是最常见事故）。
- 禁止：`display:flex` / `display:grid`、`var(--x)`、`::before/::after`、`<style>` 块、外链字体、`position:absolute/fixed`。
- 所有样式写成 `style=""` 行内属性；颜色用 hex。
- **标题里的"数字徽章 + 标题"不要用 `<table>`**——微信会渲染成带框的表格，难看。改用 `inline-block` 圆形徽章 + `vertical-align:middle` 的内联写法（见模板）。
- 正文字号 ≥ 15px，行高 ≥ 1.9，移动端无横向滚动（容器 `max-width:480px;margin:0 auto`）。

### 风格基调（区别于技术文）
- **无衬线字体**（`'PingFang SC','Microsoft YaHei'`），不用严肃宋体。
- 渐变 Hero、圆角卡片、轻投影、emoji、口语化导语和结尾，整体轻快。

### 配色（每条一色，制造活力）
| 条目 | 主色 | 浅底 |
|------|------|------|
| 1 | `#ef4444` 红 | `#fee2e2` |
| 2 | `#2563eb` 蓝 | `#dbeafe` |
| 3 | `#7c3aed` 紫 | `#ede9fe` |
| 4 | `#16a34a` 绿 | `#dcfce7` |
| 5 | `#f97316` 橙 | `#ffedd5` |

Hero 渐变：`linear-gradient(135deg,#3b5bff 0%,#6a3df0 100%)`，并写一个 hex 纯色兜底。

### 组件清单（模板里都有）
1. **Hero**：圆角渐变块 + `WEEKLY TECH` 标签 + 大标题 + 日期 + "N 条要闻 · X 分钟读完"。
2. **导语**：一句轻松开场。
3. **新闻卡片**（核心，重复 N 次）：白底 + 左侧 5px 彩色条 + 圆角 + 轻投影；标题行 = 内联圆形数字徽章 + 正式标题；正文段落，关键数字 `<strong style="color:主色">`。
4. **高亮小框**：卡内浅底圆角块，放补充信息（如身价、彩蛋）。
5. **Tips 框**：虚线边框 + 暖黄底 `#fffbeb` + 💡，放冷知识/背景。
6. **分割线**：居中细线 + ✦。
7. **结尾块**：浅灰圆角，口语化收尾 + "点在看"引导。
8. **Footer**：标签流 + 免责声明（"本文由 AI 整理自公开报道，数据以官方信息为准"）。

> 卡片标题不要带"emoji+分类词"标签（如"🤖 AI·监管"），直接用正式标题更干净——除非用户要求分类标签。

## ③ 封面图

用文章同款蓝紫渐变做 SVG 封面，再转 PNG（微信封面只收 JPG/PNG，不收 SVG）。

- 模板见 **`assets/cover-template.svg`**：2.35:1（viewBox 900×383）。含 Hero 渐变、`WEEKLY TECH` 标签、大标题、日期、副标题、火箭 + 轨道环 + 星点、底部五个彩点（呼应五条新闻配色）。
- **标准尺寸**：公众号主封面 2.35:1；可另出 1:1 方形版（缩略图/分享卡）。

### SVG 转 PNG（在沙箱里）

沙箱通常**没有 PingFang**，且自带中文字体（Droid Sans Fallback）**不含拉丁字母和数字**——直接转 PNG 会让中文或英文数字变成方框 □。必须按字符类型分别指定字体。用 **`assets/render_cover.py`**：

```bash
pip install cairosvg --break-system-packages -q
python3 assets/render_cover.py 封面.svg 封面.png 1800 766
```

脚本做的事：把根字体换成中文可用字体（Droid Sans Fallback），把纯西文元素和中英混排里的数字/字母片段单独指定西文字体（Liberation Sans / DejaVu Sans），再渲染。生成后**务必 Read 一遍 PNG 自检**有没有方框。

## 输出与交付

1. 产物全部存到用户选定的工作目录，文件名带日期，如 `本周科技要闻-YYYYMMDD.md / _wechat.html / _封面.png / _封面.svg`。
2. 用 present_files 让用户查看/下载。
3. HTML 附使用说明：
   ```
   ① Chrome 打开 → ② Ctrl+A 全选 → Ctrl+C → ③ mp.weixin.qq.com 新建图文 → Ctrl+V → ④ 手机预览后发布
   ```
4. 完成后可主动提议：用 schedule 设成每周定时生成。

## 自检清单
- [ ] HTML 有 `<meta charset="UTF-8">`
- [ ] 无 flex/grid/var/伪元素/外链字体/style 块
- [ ] 标题用内联徽章，非 table
- [ ] 所有新闻已核实时效与数字，附了 Sources
- [ ] 封面 PNG 已 Read 检查无方框
